# Schema — Query Operation

## 정의

기존 wiki/ 페이지들을 탐색·합성해 답변을 생성하는 작업.
쿼리 시점에 검색 없이 Compilation된 wiki를 읽는 것이 원칙 (RAG 금지).

## 호출 시점

| 트리거 | 비고 |
|--------|------|
| 선행 지식 조회 (관련 결정·실험 유무) | 설계 분기 전 |
| 개념·원리 정의 확인 | 수시 |
| 다음 행동 전 기존 교훈 점검 | 세션 시작 시 |
| 여러 소스 합성이 필요한 설계 질문 | 복잡도 중·상 |

## 호출 주체 분기

| 복잡도 | 주체 |
|--------|------|
| 단순 (페이지 1~2개 조회) | 에이전트 직접 수행 |
| 교차 합성 (페이지 3개+) | Sonnet 서브에이전트 위임 |
| 원리 승격 판단 포함 | Opus Advisor 호출 |
| 인간 직접 탐색 | Obsidian `Ctrl+Shift+F` 또는 graph view |

## 절차

1. **관련 wiki 페이지 탐색** — 섹션 디렉토리 탐색 또는 wikilink 그래프 네비게이션
2. **교차 참조로 답변 합성** — 단일 페이지 답변 금지; 여러 페이지 교차 참조
3. **새 synthesis 페이지 작성** (필요 시) — 새 통찰 발생 시 `concepts/` 또는 `principles/`에 신규 페이지 → Ingest 전환
4. **log.md 기록** — `## YYYY-MM-DD — [Query] <질문 요약> | <참조 페이지 목록>`

## 답변 형식

```markdown
### 질문: [질문 텍스트]

**합성 답변:**
[교차 참조 기반 답변]

**참조 페이지:**
- [[page_1]] — 해당 내용 1줄 요약
- [[page_2]] — 해당 내용 1줄 요약

**주의:** [모순 발견 또는 불확실 영역 있으면 명시]
```

## 주의사항

- Query는 wiki를 **읽기 전용** 소비. 내용 수정 필요 시 → Ingest 전환.
- 합성 결과가 기존 페이지와 모순 발견 시 → Lint 트리거 고려.
- RAG 기반 검색(벡터 유사도) 금지 — wiki 디렉토리 직접 탐색 우선.
- 답변에 없는 지식은 "현재 wiki에 없음"으로 명시 — 할루시네이션 금지.
